Import fonts from Google

We can import fonts from Google Fonts using the next <link> tag just as follows:

<link href="https://fonts.googleapis.com/css?family=FAMILY_NAME" rel="stylesheet" type="text/css">

Now, the new FAMILY_NAME font can be used.

Family names are case-sensitive and need to be wrapped in quotes if there is a space in the name.

Generic font family names are not case-sensitiveand do not need quotes because they are CSS keywords.


Fallback fonts

When writing CSS rules for font families, we can set the fallback fonts. To do that, do as it follows:

font-family: FAMILY_NAME, GENERIC_NAME;

This way, in case that FAMILY_NAME can't be reached for whichever reasons, the GENERIC_NAMEwill substitute the FAMILY_NAME


ID's importance & best practices

  1. Can be used to select and modify specific elements with JavaScript.

  2. ids attributes should be unique. Don't give more than one element the same id attribute.

  3. ids have higher specificity than a class; thus, if conflicting styles result from both class and id applied to the same element, the styles of the id will prevail.

Other CSS Selectors

The form [attr=value] can be used as another type of CSS selector. It matches and styles elements with a specific attribute=value pair.

For example:

[type='radio'] will affect to all elements with the attribute type which have a value of radio, as it could be:

<label><input type="radio" name="indoor-outdoor"> Outdoor</label>

Relative Units: em & rem

  1. em is relative to the size of its parent's font-size.

  2. rem is relative to the root -- this is to the html tag.

CSS Variables

selector {
--custom_property_name: CSS value;
}

Example:

.variables {
--main-bg-color: red;
}

Application:

element {
CSS property: var(--custom_property_name);
}

Example:

h2 {
background-color: var(--main-bg-color);
}

CSS Variables -- Notes & Best Practices

To define custom properties on the :root pseudo-class, so that it can be applied globally across your HTML document.

Custom property names are case sensitive


CSS Variables -- Fallbacks

  1. Multiple fallback values can be defined within the var() function.

  2. Fallback values aren't used to fix the browser compatibility. If the browser doesn't support CSS custom Properties, the fallback value won't help.

  3. Thus, It's just a backup for the browser which supports CSS Custom Properties to choose a different value if the given variable isn't defined or has an invalid value.

How did CSS Variables Fallbacks Works?

var(custom_property_name, fallback value)

The function only accepts two parameters, but they can be chained as follows:

selector {
background-color: var(--my-var, var(--my-background, pink));
}

Improving Compatibility with Browser Fallbacks

Another way to improve browser compatibility is to declare another more widely supported value immediately before.

Basically, instead of...

element {
background: var(--my-variable, black)
}

...we could use...

element {
background: black;
background: var(--my-variable)
}

CSS Properties: box-shadow

The box-shadow property applies one or more shadows to an element.

Values of the box-shadowproperty

  1. offset-x: how far to push the shadow horizontally from the element.
  2. offset-y: how far to push the shadow vertically from the element.
  3. blur-radius: optional
  4. spread-radius: optional
  5. color

Multiple box-shadows can be created by using commas to separate properties of each box-shadow.

Example:

element {
box-shadow: 0 10px 20px rgba(0,0,0,0.19), 0 6px 6px rgba(0,0,0,0.23)
};

CSS Box Model and relative positions

CSS Box Model: the treatment that each HTML element receive as if they were boxes.

Block-level items automatically start on a new line:

  1. Headings
  2. Paragraphs
  3. Divs

When the position of an element is set to relative, it allows you to specify how CSS should move it relative to its current position in the normal flow of the page.

It pairs with the CSS offset, which say how much to move the item away from its normal position.

CSS Offset properties are:

  • left
  • right
  • bottom
  • top

Changing an element's position to relative does not remove it from the normal flow; this is, other elements around it still behave as if that item were in its default position.

No matter the position of elements, the underlying HTML markup should be organized and make sense when read from top to bottom. This is how users with visual impairments (who rely on assistive devices like screen readers) access your content.
