Commenting HTML

In order to comment HTML, <!--***--> is used -- where *** is the comment itself.


ALT attribute for images -- best practices

If the image is purely decorative, using an empty alt attribute is a best practice.

Ideally the alt attribute should not contain special characters unless needed.


Forms -- best practices

  1. It is considered best practice to set a for attribute on the label element, with a value that matches the value of the id attribute of the input element. This allows assistive technologies to create a linked relationship between the label and the child input element. For example:

    2. <label for="color">Favorite color?
      <input id="color" type="radio" name="colors">Black
      <input id="color" type="radio" name="colors">White
    </label>

    Result:


  2. Inputs of type="radio" and type="checkbox" report their values from the value attribute in the form: name=value. For example:

    3. <label for="color">Favorite color?
      <input id="color" type="radio" name="colors" value="black">Black
      <input id="color" type="radio" name="colors" value="white">White
    </label>

    Result:

    colors:black

    or

    colors:white

Import fonts from Google

We can import fonts from Google Fonts using the next <link> tag just as follows:

<link href="https://fonts.googleapis.com/css?family=FAMILY_NAME" rel="stylesheet" type="text/css">

Now, the new FAMILY_NAME font can be used.

Family names are case-sensitive and need to be wrapped in quotes if there is a space in the name.


Fallback fonts

When writing CSS rules for font families, we can set the fallback fonts. To do that, do as it follows:

font-family: FAMILY_NAME, GENERIC_NAME;

This way, in case that FAMILY_NAME can't be reached for whichever reasons, the GENERIC_NAMEwill substitute the FAMILY_NAME
